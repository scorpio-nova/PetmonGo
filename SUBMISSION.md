## 项目说明 `project_description`（按 4 点结构，第 1、4 点详细）

```
1. 解决什么问题 / 真实痛点是什么？
Petmon Go 是一张"街区宠物地图"，集合爱宠人士关心的话题事件：每只猫狗有档案和行为方式记录，每起安全事件有位置和类型化提醒，让爱猫人、宠物主和普通居民共享共建同一份街区宠物信息，并体验收集游戏的乐趣。

2. 目标用户是谁？

社区流浪猫喂养者/志愿者、散养宠物的主人、遛狗居民，以及关心社区安全（怕被抓咬、反对虐猫）的普通住户。以小区/街道为单位的熟人半熟人社区。

3. 核心方案 / 技术实现是什么？

产品形态是"宠物版 Pokémon Go"：地图 + 图鉴 + 偶遇打卡。核心闭环为：拍照识别 → 匹配已知宠物（进档案上传相遇）或建档新宠物 → 收集图鉴与今日踪迹。特色机制包括：野生/家养/天使三类宠物 Tag，"天使宠物"纪念卡（彩虹纪念框 + 供奉小鱼干/小骨头），四类安全事件（虐猫/丢失/抓人/咬人）的类型化文案与地图标注。

技术上最核心的是AI时代艺术家手绘的 ui，以及宠物识别与匹配建档功能：(1)前端为组件化 HTML/JS 单页应用，手绘风 UI（稚拙涂鸦风 + 手写字体）。(2)用 transformers.js 在浏览器里跑 CLIP（Xenova/clip-vit-base-patch32），把用户上传的照片编码成 512 维向量，与离线预计算的图库向量（tools/build-embeddings.mjs 生成 embeddings.json）做余弦相似度，按宠物取最高分排序返回 Top3——无需任何后端和 GPU，模型 CDN 加载后本地缓存。

4. 现场 36 小时完成了哪些功能？评审如何查看成果？

现场完成（全部代码 commit 均在比赛周期内，可查 git log）：
- 全部 10+ 页面：地图首页（宠物/事件点位）、探索页（宠物日志/天使故事/地点评价/遛宠路线四栏目）、图鉴（新动态红点）、宠物详情（今日踪迹时间线）、上传相遇、宠物识别、新宠物建档、安全事件发布与详情、天使纪念卡、消息中心；
- 完整状态流：偶遇上传写入踪迹、通知数据驱动、跨页返回链、图鉴已读/未读红点；
- CLIP 宠物识别全链路：图库照片离线建 embedding，识别页放入照片自动匹配，动态渲染 Top3 相似度卡片（命中图鉴宠物可一键进档案，未收录则引导建档）；

评审查看方式：
1. 
2. clone 仓库后在 project/ 目录运行 python3 -m http.server，浏览器打开 petmon go.dc.html 即可完整体验（识别功能首次使用需联网下载约 30MB CLIP 模型）；也可直接看仓库内 tools/sim-recognize.png 的识别结果实测截图。
```
